# Word2Vec — Semantic Aspects

## Overview

**Word2Vec** (Mikolov et al., 2013) is a **prediction-based** (as opposed to count-based) embedding method that learns dense vector representations of words by training a shallow neural network on a simple auxiliary task — predicting context. The key insight: *the task itself doesn't matter as much as the fact that solving it forces the model to organize words by meaning.*

> **Conceptual framing**: Word2Vec is not really "about" prediction. Prediction is just the training signal — a means to an end. The real product is the **weight matrix learned along the way**, which becomes the embedding table.


## Architecture Deep Dive

Word2Vec is a **shallow, 2-layer neural network** (no deep stack, no attention — this predates Transformers by 4 years). It has:

```
Input Layer (one-hot, size = |V|)
        ↓
Hidden Layer (size = embedding dimension, e.g., 300) ← this becomes the embedding
        ↓
Output Layer (softmax over vocabulary, size = |V|)
```

The **hidden layer weight matrix** (size `|V| × d`) *is* the embedding table. After training, you throw away the output layer entirely — you only keep this matrix.

### CBOW (Continuous Bag of Words)

Predicts a **target word** from the **average of its context word vectors**.

```
Context window (size=2): ["the", "cat", ___, "on", "the"]
                                     ↑
                              Target: "sat"

Input: average(vec("the"), vec("cat"), vec("on"), vec("the"))
Output: softmax prediction → "sat"
```

- Averages context vectors → smooths out noise
- **Faster to train**, better for frequent words
- Best when context matters more than any single word

### Skip-gram

The reverse: predicts **context words** from a **single target word**.

The Skip‑gram model in Word2Vec is a training approach where the system learns word embeddings by predicting the context words around a given target word. In practice, this means that for every word in a sentence, the model tries to guess which words appear nearby. For example, in the sentence “The cat sat on the mat”, if the target word is “cat” and the context window size is 2, the model generates training pairs like (“cat”, “the”), (“cat”, “sat”), and (“cat”, “on”). Each of these pairs is used to update the embedding vectors so that words appearing in similar contexts end up with similar representations.

This training process is repeated across the entire corpus, so words like “cat” and “dog” — which often appear in similar contexts such as “pet”, “mat”, or “rug” — gradually move closer together in the embedding space. That’s why Skip‑gram embeddings capture semantic similarity: they reflect the distribution of contexts in which words occur.

```
Target: "sat"
                        ↓
Predict each independently:
  P("the"  | "sat")
  P("cat"  | "sat")
  P("on"   | "sat")
  P("the"  | "sat")
```

- Doesn't average — treats each (target, context) pair as a separate training example
- **Slower but higher quality**, especially for **rare words** (each rare word gets more individual training signal)
- This is the version most production systems historically used

**Rule of thumb**: `Skip-gram > CBOW` for embedding quality; `CBOW > Skip-gram` for training speed.


## The Training Objective (Math)

For Skip-gram, given a sequence of words `w1, w2, ..., wT`, the objective is to maximize the average log probability:

$$
\frac{1}{T} \sum_{t=1}^{T} \sum_{-c \leq j \leq c,\ j \neq 0} \log P(w_{t+j} \mid w_t)
$$

Where `c` is the context window size, and:

$$
P(w_O \mid w_I) = \frac{\exp(v'_{w_O} \cdot v_{w_I})}{\sum_{w=1}^{|V|} \exp(v'_w \cdot v_{w_I})}
$$

**The problem**: that denominator sums over the *entire vocabulary* — for a 100,000-word vocabulary, that's 100,000 dot products **per single training step**. This is computationally infeasible at scale, which is exactly why two clever approximations were introduced:

### 1. Negative Sampling

Instead of updating weights for *all* vocabulary words on every step, only update:
- The actual context word (positive example)
- A small handful of randomly sampled "negative" words (typically 5–20) that *don't* appear in the context

```
Target: "sat", Real context word: "cat"     → label = 1 (positive pair)
Target: "sat", Random word: "pizza"         → label = 0 (negative sample)
Target: "sat", Random word: "quantum"       → label = 0 (negative sample)
```

This turns an expensive `|V|`-way softmax into a handful of cheap binary classification problems — the single biggest reason Word2Vec could scale to billion-word corpora.

Negative words are sampled based on a modified unigram frequency:

$$
P(w) = \frac{f(w)^{3/4}}{\sum_{w'} f(w')^{3/4}}
$$

The `3/4` power dampens the dominance of very frequent words (like "the") while still slightly favoring frequent words over rare ones as negative samples.

### 2. Hierarchical Softmax

An alternative to negative sampling: organize vocabulary into a **binary tree** (typically a Huffman tree, so frequent words get shorter paths), turning the `|V|`-way prediction into a sequence of `O(log|V|)` binary decisions down the tree instead of one massive softmax.

| Method | Best for |
|---|---|
| Negative Sampling | Large corpora, frequent words, general use |
| Hierarchical Softmax | Rare words, smaller vocabularies |


## Core Concepts

| **Concept** | **What it Means** | **Example** |
|-------------|-------------------|-------------|
| **Distributional Hypothesis** | Words with similar meanings appear in similar contexts | "cat" and "dog" often occur near "pet," "animal," "food" |
| **CBOW** | Predicts a word given surrounding context (averaged) | Context: "the ___ sat on the mat" → predicts "cat" |
| **Skip-gram** | Predicts surrounding context words given a target word | Target: "cat" → predicts "sat," "mat" |
| **Negative Sampling** | Approximates full softmax with a few random negatives | Only update weights for ~5-20 sampled words, not entire vocab |
| **Embedding Space** | Learned vectors form a space where distance reflects meaning | king − man + woman ≈ queen |
| **Semantic Similarity** | Cosine similarity measures closeness of meaning | cos(cat, dog) > cos(cat, car) |
| **Subsampling of Frequent Words** | Downsample very common words (the, is, a) during training | Prevents "the" from dominating every context window |


## Key Hyperparameters (Often Overlooked)

| Hyperparameter | Typical Value | Effect |
|---|---|---|
| **Embedding dimension (`d`)** | 100–300 | Higher = more expressive, but diminishing returns + more data needed |
| **Window size (`c`)** | 5–10 | Small window → captures syntactic similarity; large window → captures topical/semantic similarity |
| **Min word count** | 5 | Words appearing fewer times are dropped (too little signal to learn a good vector) |
| **Negative samples** | 5–20 | More negatives = better quality but slower training |
| **Epochs** | 5–15 | Passes over the corpus |
| **Subsampling threshold** | ~1e-5 | Frequent words (the, is, of) are randomly dropped from training pairs to reduce noise |

**Research insight on window size**: this parameter alone changes *what kind* of similarity gets learned:
- Small window (2-3) → captures **syntactic/functional similarity** (e.g., "good" ~ "bad," both adjectives in similar slots)
- Large window (8-10) → captures **topical similarity** (e.g., "good" ~ "excellent," "wonderful" — thematically related)


## Workflow

1. **Prepare Corpus** — collect large text data (Wikipedia, news, domain-specific text)
2. **Tokenize + Clean** — lowercase, remove punctuation/rare tokens (implementation detail, but affects vocabulary quality significantly)
3. **Choose Architecture** — CBOW (speed) vs Skip-gram (quality, rare words)
4. **Choose Training Approximation** — Negative Sampling (common) vs Hierarchical Softmax
5. **Train Model** — network learns via gradient descent (typically SGD) over the co-occurrence signal
6. **Extract Embeddings** — discard output layer; keep the hidden layer weight matrix as the embedding table
7. **Evaluate** — analogy tasks, similarity benchmarks (e.g., WordSim-353), or downstream task performance
8. **Use Embeddings** — clustering, retrieval, analogy, or as frozen/fine-tuned input to downstream models


## Examples

```python
from gensim.models import Word2Vec

# Train
sentences = [["the", "cat", "sat", "on", "the", "mat"],
             ["the", "dog", "sat", "on", "the", "log"]]

model = Word2Vec(
    sentences,
    vector_size=300,     # embedding dimension
    window=5,             # context window size
    min_count=1,          # keep all words (toy example)
    sg=1,                 # 1 = skip-gram, 0 = CBOW
    negative=10,           # negative sampling count
    epochs=10
)

# Similarity
model.wv.similarity("cat", "dog")   # High similarity
model.wv.similarity("cat", "car")   # Low similarity

# Analogy
model.wv.most_similar(positive=["king", "woman"], negative=["man"])
# Output ≈ "queen"

# Nearest neighbors
model.wv.most_similar("computer")
# Output ≈ ["software", "technology", "machine", ...]

# Raw vector
model.wv["cat"]   # → dense 300-dim numpy array
```


## Strengths vs Limitations

| **Strengths** | **Limitations** |
|---------------|-----------------|
| Captures semantic similarity | One vector per word — no context sensitivity |
| Efficient to train (thanks to negative sampling) | Struggles with polysemy (e.g., "bank") |
| Supports linear analogies | Requires large corpus for quality vectors |
| Dense, low-dimensional vectors | Static embeddings — same vector every time, unlike BERT |
| Simple architecture, easy to train from scratch on domain data | Out-of-vocabulary words get no vector at all (fixed by fastText's subword approach) |
| Fast inference (just a lookup table) | Encodes and can amplify societal biases present in training text |


## Word2Vec vs GloVe vs fastText (Quick Distinctions)

| Method | Learns From | Key Idea |
|---|---|---|
| **Word2Vec** | Local context windows (predictive) | Predict neighboring words |
| **GloVe** | Global co-occurrence matrix (count-based) | Factorize log co-occurrence counts directly |
| **fastText** | Character n-grams + local context | Represent words as sums of subword vectors → handles OOV/rare/morphological words |


## Applications

- **Semantic search** — embedding-based similarity instead of exact keyword match
- **Clustering** — grouping semantically related words/documents
- **Recommendation systems** — item2vec-style extensions using the same skip-gram machinery
- **RAG pipelines** — early/lightweight retrieval components (though modern RAG typically uses contextual retrieval embeddings like E5/BGE instead)
- **Text classification** — as frozen or fine-tuned input features
- **Bias auditing research** — studying what societal associations get encoded (e.g., gender-occupation bias analysis)


## Why Word2Vec Mattered (Research Significance)

Word2Vec's real contribution wasn't embeddings themselves (count-based methods like LSA already produced dense vectors) — it was proving that:

1. **A cheap, shallow, predictive objective** could produce embeddings *as good or better* than expensive matrix factorization methods
2. **Negative sampling** made training scale to billion-word corpora on commodity hardware — a systems-level insight as much as an ML one
3. **Linear structure emerges from co-occurrence statistics alone** — no one designed the model to encode "gender" or "royalty" as directions; this was an *emergent* property, which sparked an entire research direction into interpretability and structure of embedding spaces

This is what set the stage for everything that followed: if a shallow network could learn this much just from co-occurrence, what could a *deep* model learn from co-occurrence **plus context**? That question directly leads to ELMo → BERT → Transformers.
