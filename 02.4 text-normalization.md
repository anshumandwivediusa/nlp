# Text Normalization

## Overview

**Text Normalization** is the process of converting text into a consistent, standardized format while preserving its meaning. It reduces variations in text caused by differences in capitalization, spelling, punctuation, abbreviations, contractions, and formatting.

Normalization improves the consistency of textual data, making it easier for NLP models and machine learning algorithms to process and analyze.

It is one of the most important steps in **Text Preprocessing** and is commonly performed after **Unicode & Encoding Handling** and **Text Cleaning**.



# Why is Text Normalization Important?

Natural language can be written in many different ways while conveying the same meaning.

For example:

```text
Microsoft
MICROSOFT
microsoft
```

Without normalization, these may be treated as three different words by traditional NLP systems.

Normalization helps to:

* Reduce vocabulary size
* Improve text consistency
* Increase search accuracy
* Improve model performance
* Simplify feature extraction
* Reduce data sparsity

# Common Text Normalization Techniques

## 1. Case Folding

Converts text to a consistent letter case, typically lowercase.

### Example

| Before         | After          |
| -------------- | -------------- |
| `Microsoft AI` | `microsoft ai` |
| `HELLO World`  | `hello world`  |

### Benefits

* Reduces duplicate words.
* Simplifies text comparison.
* Improves search indexing.



## 2. Contraction Expansion

Expands shortened forms into their complete words.

### Example

| Before  | After    |
| ------- | -------- |
| `don't` | `do not` |
| `I'm`   | `I am`   |
| `it's`  | `it is`  |
| `can't` | `cannot` |

### Benefits

* Improves semantic understanding.
* Useful for sentiment analysis and question answering.



## 3. Whitespace Normalization

Removes unnecessary spaces, tabs, and line breaks.

### Example

Before

```text
Hello        World
```

After

```text
Hello World
```



## 4. Punctuation Normalization

Standardizes punctuation by removing duplicates or converting inconsistent punctuation.

### Example

| Before      | After    |
| ----------- | -------- |
| `Hello!!!`  | `Hello!` |
| `Wait.....` | `Wait.`  |

> **Note:** In many modern NLP and LLM applications, punctuation is preserved because it carries semantic and syntactic information.



## 5. Number Normalization

Converts numerical values into a consistent format.

### Example

| Before  | After  |
| ------- | ------ |
| `1,000` | `1000` |
| `15th`  | `15`   |

Applications include financial NLP, analytics, and information extraction.



## 6. Date & Time Normalization

Converts dates into a standardized representation.

### Example

| Before          | After        |
| --------------- | ------------ |
| `15 March 2025` | `2025-03-15` |
| `03/15/2025`    | `2025-03-15` |

Useful for event extraction, scheduling, and document processing.



## 7. Unicode Normalization

Converts visually identical Unicode characters into a consistent representation.

### Example

Both strings below look identical:

```text
Café
```

```text
Café
```

After Unicode normalization, both are stored using the same Unicode representation.



## 8. Spelling Normalization

Corrects spelling mistakes or standardizes spelling variants.

### Example

| Before    | After                                            |
| --------- | ------------------------------------------------ |
| `recieve` | `receive`                                        |
| `colour`  | `color` *(if standardizing to American English)* |

Applications:

* Search engines
* OCR
* Chatbots



## 9. Abbreviation Expansion

Expands abbreviations into their complete forms.

### Example

| Before  | After                                        |
| ------- | -------------------------------------------- |
| `Dr.`   | `Doctor`                                     |
| `Dept.` | `Department`                                 |
| `AI`    | `Artificial Intelligence` *(task-dependent)* |



## 10. Character Normalization

Standardizes repeated or elongated characters.

### Example

| Before        | After                                              |
| ------------- | -------------------------------------------------- |
| `sooooo good` | `soo good` *(or `so good`, depending on the task)* |
| `Yessss!!`    | `Yes!`                                             |

Useful for social media and user-generated content.



# Summary of Techniques

| Technique                 | Description                                     | Example                              |
| ------------------------- | ----------------------------------------------- | ------------------------------------ |
| Case Folding              | Converts text to lowercase or a consistent case | `HELLO` → `hello`                    |
| Contraction Expansion     | Expands contractions                            | `don't` → `do not`                   |
| Whitespace Normalization  | Removes extra spaces and tabs                   | `"Hello    World"` → `"Hello World"` |
| Punctuation Normalization | Standardizes punctuation                        | `Hello!!!` → `Hello!`                |
| Number Normalization      | Standardizes numeric formats                    | `1,000` → `1000`                     |
| Date & Time Normalization | Converts dates to a consistent format           | `15 March 2025` → `2025-03-15`       |
| Unicode Normalization     | Standardizes Unicode characters                 | `Café` → `Café`                     |
| Spelling Normalization    | Corrects spelling mistakes                      | `recieve` → `receive`                |
| Abbreviation Expansion    | Expands abbreviations                           | `Dr.` → `Doctor`                     |
| Character Normalization   | Removes repeated characters                     | `soooo` → `soo`                      |



# Traditional NLP vs Modern LLMs

| Technique                   | Traditional NLP   | Modern LLMs         |
| --------------------------- | ----------------- | ------------------- |
| Case Folding                | ✅ Common          | ⚠️ Task-dependent   |
| Contraction Expansion       | ✅ Common          | ⚠️ Optional         |
| Unicode Normalization       | ✅ Required        | ✅ Required          |
| Whitespace Normalization    | ✅ Required        | ✅ Required          |
| Number & Date Normalization | ✅ Common          | ⚠️ Task-dependent   |
| Spelling Correction         | ✅ Common          | ⚠️ Optional         |
| Punctuation Normalization   | ✅ Often performed | ❌ Usually preserved |

Modern Transformer-based models generally require **less aggressive normalization** because they learn contextual representations. However, Unicode normalization, whitespace normalization, and consistent encoding remain essential preprocessing steps.



# Best Practices

* Normalize Unicode before any linguistic processing.
* Use UTF-8 encoding throughout the pipeline.
* Apply case folding only if it aligns with the task.
* Preserve punctuation when using Transformer-based models unless there is a clear reason to remove it.
* Standardize dates and numbers when they are important for downstream applications.
* Use the same normalization pipeline during both training and inference.



# Summary

Text Normalization converts raw text into a consistent and standardized representation while preserving its meaning. It reduces unnecessary variation in language, improves data quality, and enhances the performance of NLP systems.

A typical normalization pipeline includes:

1. Case Folding
2. Contraction Expansion
3. Whitespace Normalization
4. Punctuation Normalization
5. Number Normalization
6. Date & Time Normalization
7. Unicode Normalization
8. Spelling Normalization
9. Abbreviation Expansion
10. Character Normalization

Effective text normalization is a key foundation for tokenization, feature extraction, semantic analysis, machine learning, and modern Large Language Models (LLMs).
