# Syntactic Analysis

## Overview

**Syntactic Analysis**, also known as **Parsing**, is the process of analyzing the grammatical structure of a sentence. It determines how words are organized and how they relate to one another according to the rules of a language.

Unlike **Lexical Analysis**, which focuses on individual words, syntactic analysis examines the relationships between words to understand the sentence's structure.

Syntactic analysis is a fundamental stage of Natural Language Processing (NLP) and serves as a bridge between lexical processing and semantic understanding.



# Why is Syntactic Analysis Important?

Understanding the grammatical structure of a sentence helps NLP systems:

* Identify the role of each word.
* Understand subject–verb–object relationships.
* Detect phrases and clauses.
* Improve language understanding.
* Support downstream NLP tasks such as machine translation, question answering, and information extraction.

# Major Techniques

## 1. Part-of-Speech (POS) Tagging

### Overview

Part-of-Speech (POS) Tagging assigns a grammatical category to each word in a sentence, such as noun, verb, adjective, or preposition.

It helps NLP systems understand the grammatical function of every token.

### Example

| Word      | POS Tag     |
| --------- | ----------- |
| John      | Proper Noun |
| submitted | Verb        |
| the       | Determiner  |
| project   | Noun        |
| report    | Noun        |
| to        | Preposition |
| Microsoft | Proper Noun |
| yesterday | Adverb      |

### Applications

* Grammar checking
* Machine translation
* Information extraction
* Text-to-speech
* Parsing



## 2. Dependency Parsing

### Overview

Dependency Parsing identifies grammatical relationships between words by determining which words depend on others.

Each relationship consists of:

* **Head** (governing word)
* **Dependent** (related word)

### Example

```text
submitted
├── Subject → John
├── Object → report
├── Modifier → project
├── Preposition → to
│       └── Object → Microsoft
└── Time → yesterday
```

Dependency parsing answers questions such as:

* Who performed the action?
* What was acted upon?
* Where did it happen?
* When did it happen?

### Applications

* Information extraction
* Question answering
* Relation extraction
* Semantic search



## 3. Constituency Parsing

### Overview

Constituency Parsing divides a sentence into hierarchical phrases based on grammar rules.

Instead of focusing on word-to-word relationships, it groups words into meaningful constituents.

### Example

```text
(S
   (NP John)
   (VP
      submitted
      (NP the project report)
      (PP to Microsoft)
      (ADVP yesterday)
   )
)
```

Common phrase types include:

| Phrase | Meaning              |
| ------ | -------------------- |
| NP     | Noun Phrase          |
| VP     | Verb Phrase          |
| PP     | Prepositional Phrase |
| ADJP   | Adjective Phrase     |
| ADVP   | Adverb Phrase        |

### Applications

* Grammar analysis
* Sentence generation
* Machine translation
* Text summarization



## 4. Chunking (Shallow Parsing)

### Overview

Chunking groups nearby words into meaningful phrases without constructing a complete parse tree.

Unlike constituency parsing, chunking identifies only major phrase boundaries.

### Example

```text
[NP John]
[VP submitted]
[NP the project report]
[PP to Microsoft]
[ADVP yesterday]
```

### Applications

* Named Entity Recognition (NER)
* Information extraction
* Fast text processing
* Rule-based NLP systems



# Comparison of Techniques

| Technique            | Focus                             | Output                          |
| -------------------- | --------------------------------- | ------------------------------- |
| POS Tagging          | Grammatical category of each word | Noun, Verb, Adjective, etc.     |
| Dependency Parsing   | Relationships between words       | Subject, Object, Modifier, etc. |
| Constituency Parsing | Hierarchical phrase structure     | Parse Tree                      |
| Chunking             | Phrase identification             | Noun Phrases, Verb Phrases      |



# Traditional vs Modern NLP

## Traditional NLP

Syntactic analysis is performed explicitly using dedicated parsers and statistical models.

Common algorithms include:

* Hidden Markov Models (HMM)
* Conditional Random Fields (CRF)
* Probabilistic Context-Free Grammars (PCFG)
* Transition-Based Parsers
* Graph-Based Parsers



## Deep Learning

Neural networks improve parsing accuracy by learning syntactic patterns from data.

Examples include:

* BiLSTM Parsers
* Neural Dependency Parsers
* Neural Constituency Parsers



## Transformer Models

Models such as BERT and GPT do not explicitly perform parsing during inference. Instead, they learn syntactic relationships implicitly through self-attention mechanisms.

Although no parse tree is generated internally, these models capture rich grammatical information that can be leveraged for downstream tasks.

# Summary

Syntactic Analysis examines the grammatical structure of a sentence by identifying the role and relationships of words.

The primary techniques include:

* **Part-of-Speech (POS) Tagging** – Assigns grammatical categories to words.
* **Dependency Parsing** – Identifies grammatical dependencies between words.
* **Constituency Parsing** – Groups words into hierarchical phrases.
* **Chunking (Shallow Parsing)** – Detects major phrase boundaries without building a complete parse tree.

Together, these techniques provide the structural understanding required for higher-level semantic analysis and many downstream NLP applications.
