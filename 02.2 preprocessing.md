# Text Preprocessing

## Overview

**Text Preprocessing** is the process of cleaning, standardizing, and transforming raw text into a structured format that can be efficiently processed by Natural Language Processing (NLP) models and machine learning algorithms.

Raw text collected from websites, documents, social media, emails, OCR systems, or PDFs is often noisy and inconsistent. Text preprocessing removes this noise while preserving meaningful information, improving the quality of downstream NLP tasks.

It is the **first stage of almost every NLP pipeline** and has a significant impact on the accuracy and performance of applications such as:

* Text Classification
* Sentiment Analysis
* Named Entity Recognition (NER)
* Machine Translation
* Semantic Search
* Question Answering
* Document Summarization
* Retrieval-Augmented Generation (RAG)
* Large Language Models (LLMs)

---

# Why is Text Preprocessing Important?

Raw text often contains inconsistencies that make it difficult for NLP models to process effectively.

Common issues include:

* Inconsistent capitalization
* HTML and XML tags
* URLs and hyperlinks
* Email addresses
* Emojis and emoticons
* Special characters
* Extra whitespace
* Misspellings
* Different character encodings
* Duplicate characters
* OCR errors
* Noise from web scraping
* Hidden Unicode control characters

Without preprocessing, these inconsistencies can lead to:

* Poor tokenization
* Duplicate or incorrect embeddings
* Lower search accuracy
* Reduced model performance
* Incorrect information extraction

# Core Preprocessing Techniques

| Technique                            | Description                                                                            | Typical Use Cases                     |
| ------------------------------------ | -------------------------------------------------------------------------------------- | ------------------------------------- |
| **Unicode & Encoding Handling**      | Converts text to a consistent encoding (UTF-8) and normalizes Unicode characters.      | OCR, multilingual NLP, web scraping   |
| **Hidden Control Character Removal** | Removes invisible Unicode control characters such as Zero Width Spaces and BOM.        | PDFs, OCR, copied text                |
| **Text Cleaning**                    | Removes HTML, URLs, emails, emojis, special characters, and unnecessary whitespace.    | Data cleaning, search indexing        |
| **Text Normalization**               | Standardizes text by lowercasing, expanding contractions, and normalizing punctuation. | Chatbots, sentiment analysis          |
| **Sentence Segmentation**            | Splits documents into individual sentences.                                            | Summarization, translation            |
| **Tokenization**                     | Splits text into words, subwords, or tokens.                                           | All NLP pipelines and LLMs            |
| **Stop Word Removal** *(Optional)*   | Removes common words with limited semantic value.                                      | Search engines, information retrieval |
| **Stemming** *(Optional)*            | Reduces words to their root using simple rules.                                        | Search indexing, keyword matching     |
| **Lemmatization** *(Optional)*       | Converts words to their dictionary base form using grammar.                            | Semantic search, question answering   |

# Hidden Control Characters

Hidden control characters are invisible Unicode characters that control formatting, spacing, or text direction. Although they are not visible, they can negatively affect tokenization, searching, embeddings, and text comparison.

## Common Hidden Control Characters

| Character             | Unicode  | Description                                   | Example                |
| --------------------- | -------- | --------------------------------------------- | ---------------------- |
| New Line              | `\n`     | Starts a new line                             | `Hello\nWorld`         |
| Carriage Return       | `\r`     | Moves the cursor to the beginning of the line | Windows text files     |
| Tab                   | `\t`     | Horizontal tab                                | Tab-separated values   |
| Form Feed             | `\f`     | Page break                                    | PDF extraction         |
| Vertical Tab          | `\v`     | Vertical spacing                              | Legacy text            |
| Null Character        | `\0`     | Empty byte                                    | Binary files           |
| Zero Width Space      | `U+200B` | Invisible separator                           | Hidden between words   |
| Zero Width Non-Joiner | `U+200C` | Prevents character joining                    | Persian, Arabic        |
| Zero Width Joiner     | `U+200D` | Joins characters                              | Emoji sequences        |
| Byte Order Mark (BOM) | `U+FEFF` | Indicates Unicode encoding                    | Beginning of UTF files |
| Left-to-Right Mark    | `U+200E` | Controls text direction                       | Mixed-language text    |
| Right-to-Left Mark    | `U+200F` | Controls text direction                       | Arabic/Hebrew text     |

### Example

Raw text

```text
Hello​World
```

Actual representation

```text
Hello<U+200B>World
```

After preprocessing

```text
HelloWorld
```

# Traditional NLP vs Modern LLMs

| Technique             | Traditional NLP | Modern LLMs       |
| --------------------- | --------------- | ----------------- |
| Unicode Handling      | ✅ Required      | ✅ Required        |
| Text Cleaning         | ✅ Required      | ✅ Required        |
| Text Normalization    | ✅ Required      | ✅ Required        |
| Sentence Segmentation | ✅ Common        | ✅ Common          |
| Tokenization          | ✅ Required      | ✅ Required        |
| Stop Word Removal     | ✅ Common        | ❌ Usually avoided |
| Stemming              | ✅ Common        | ❌ Rarely used     |
| Lemmatization         | ✅ Common        | ⚠️ Task-dependent |

Modern Transformer-based models generally preserve more information than traditional NLP pipelines. As a result, aggressive preprocessing (such as stop-word removal and stemming) is often unnecessary and may even reduce model performance.

---

# Best Practices

* Convert text to a consistent Unicode encoding (UTF-8).
* Normalize Unicode characters (NFC or NFKC).
* Remove hidden control characters and Byte Order Marks (BOM).
* Remove HTML, scripts, and unnecessary markup.
* Clean URLs, emails, and unwanted symbols when appropriate.
* Normalize whitespace and punctuation.
* Preserve meaningful punctuation when it contributes to semantics.
* Apply stemming or lemmatization only if required by the downstream task.
* Use the same preprocessing pipeline during both training and inference.

# Summary

Text Preprocessing is the foundation of every NLP system. It transforms noisy, unstructured text into a clean and standardized representation, enabling more accurate linguistic analysis and machine learning.

A typical preprocessing workflow includes:

1. Unicode & Encoding Handling
2. Hidden Control Character Removal
3. Text Cleaning
4. Text Normalization
5. Sentence Segmentation
6. Tokenization
7. Stop Word Removal *(optional)*
8. Stemming *(optional)*
9. Lemmatization *(optional)*

A well-designed preprocessing pipeline improves the quality of tokenization, embeddings, information retrieval, semantic understanding, and overall model performance.
