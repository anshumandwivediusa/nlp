# Unicode & Encoding Handling

## Overview

**Unicode & Encoding Handling** is the process of ensuring that text is correctly decoded, encoded, and normalized before any NLP processing begins.

Text can originate from multiple sources such as websites, PDFs, databases, emails, OCR systems, and social media, each of which may use different character encodings. If the encoding is interpreted incorrectly, the text may become corrupted, resulting in unreadable characters, failed tokenization, incorrect embeddings, and poor model performance.

Unicode handling ensures that text is represented consistently, while encoding handling ensures that the bytes are decoded using the correct character set.

## Why is it Important?

Different systems store text using different encodings, such as:

* UTF-8
* UTF-16
* UTF-32
* ASCII
* ISO-8859-1 (Latin-1)
* Windows-1252

Without proper handling, the same text may appear differently or become unreadable.

## Unicode Normalization

Some characters can be represented in multiple ways.

### Example

These two strings look identical:

```text
Café
```

```text
Café
```

The second version consists of:

```text
e + COMBINING ACUTE ACCENT
```

Although visually identical, they have different Unicode representations.

Unicode normalization converts both into a consistent form, making comparisons, searching, and tokenization reliable.

Common normalization forms include:

* NFC (Canonical Composition)
* NFD (Canonical Decomposition)
* NFKC (Compatibility Composition)
* NFKD (Compatibility Decomposition)

For most NLP applications, **NFC** or **NFKC** is recommended.

---

## Hidden Unicode Characters

Unicode handling also removes or standardizes invisible characters such as:

* Zero Width Space (`U+200B`)
* Zero Width Joiner (`U+200D`)
* Byte Order Mark (BOM) (`U+FEFF`)
* Left-to-Right Mark (`U+200E`)
* Right-to-Left Mark (`U+200F`)

These characters can silently affect searching, tokenization, and embeddings.

---

## Best Practices

* Decode text using UTF-8 whenever possible.
* Normalize Unicode using NFC or NFKC.
* Remove hidden control characters.
* Remove Byte Order Marks (BOM).
* Standardize line endings.
* Validate text before downstream processing.
