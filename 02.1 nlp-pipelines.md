# NLP Pipelines
Natural Language Processing (NLP) enables computers to understand, analyze, interpret, and generate human language. It combines techniques from linguistics, machine learning, deep learning, and Artificial Intelligence (AI) to transform unstructured text into meaningful information.

An NLP system typically follows a pipeline where raw text is progressively processed, analyzed, and transformed into structured representations before being used for prediction or text generation.

A complete NLP Pipeline represents the sequence of stages that transform raw text into meaningful insights or generated responses. The exact pipeline varies between traditional NLP, machine learning-based NLP, and modern LLM/RAG systems, but the overall progression remains similar.

**Rule of thumb:**

* **Text Preprocessing** = Cleaning and preparing the text.
* **Lexical Analysis** = Breaking prepared text into meaningful lexical units (tokens) and normalizing them for further linguistic analysis.


## 1. Preprocessing

Text Preprocessing is the process of cleaning, standardizing, and transforming raw text into a structured format that can be efficiently processed by NLP models and machine learning algorithms.

It is the first stage of almost every NLP pipeline and significantly impacts the accuracy and performance of downstream tasks such as text classification, semantic search, sentiment analysis, question answering, and Retrieval-Augmented Generation (RAG).

### Why is Preprocessing Important?

Raw text often contains:
    - Inconsistent capitalization   
    - HTML tags   
    - URLs   
    - Emojis   
    - Extra whitespace   
    - Misspellings   
    - Different character encodings   
    - Punctuation   
    - Duplicate characters   
    - Noise from web scraping or OCR
    - Hidden Crontrol Characters

    
    | **Character**         | **Unicode** | **Description**            | **Example**              |
    | --------------------- | ----------- | -------------------------- | ------------------------ |
    | New Line              | `\n`        | Starts a new line          | `Hello\nWorld`           |
    | Carriage Return       | `\r`        | Moves cursor to line start | Windows text files       |
    | Tab                   | `\t`        | Horizontal tab             | Columns in text          |
    | Form Feed             | `\f`        | Page break                 | PDFs                     |
    | Vertical Tab          | `\v`        | Vertical spacing           | Legacy text              |
    | Null Character        | `\0`        | Empty byte                 | Binary files             |
    | Zero Width Space      | `U+200B`    | Invisible separator        | Appears between words    |
    | Zero Width Non-Joiner | `U+200C`    | Prevents character joining | Persian, Arabic scripts  |
    | Zero Width Joiner     | `U+200D`    | Joins characters           | Emoji sequences          |
    | Byte Order Mark (BOM) | `U+FEFF`    | Indicates Unicode encoding | Beginning of UTF files   |
    | Left-to-Right Mark    | `U+200E`    | Controls text direction    | Mixed-language documents |
    | Right-to-Left Mark    | `U+200F`    | Controls text direction    | Arabic/Hebrew text       |

Text preprocessing removes or standardizes these inconsistencies.

## 2. Lexical Analysis
_Lexical meaning is the meaning of an individual word as defined in a dictionary (lexicon)._
**Lexical Analysis** is the process of analyzing raw text at the **word (lexical) level**. It converts unstructured text into meaningful lexical units (tokens) while preparing the text for syntactic and semantic analysis.

In a typical NLP pipeline, lexical analysis comes **after basic text cleaning** and **before syntactic analysis**.

### What Does Lexical Analysis Include?

| **Technique**                | **Purpose**                                                    |
| ---------------------------- | -------------------------------------------------------------- |
| Tokenization                 | Split text into tokens (words, subwords, punctuation).         |
| Text Normalization           | Standardize text (lowercase, expand contractions, etc.).       |
| Stop Word Removal            | Remove common words that add little meaning (traditional NLP). |
| Stemming                     | Reduce words to their root form using rules.                   |
| Lemmatization                | Convert words to their dictionary form.                        |
| Lexicon Lookup               | Match words against a dictionary or vocabulary.                |
| Handling Unknown Words (OOV) | Manage words not found in the vocabulary.                      |

## Difference Between Text Preprocessing and Lexical Analysis

| **Text Preprocessing**                           | **Lexical Analysis**                                            |
| ------------------------------------------------ | --------------------------------------------------------------- |
| Cleans and standardizes raw text.                | Analyzes text at the word/token level.                          |
| Removes noise (HTML, URLs, emojis, punctuation). | Produces lexical units (tokens) for further analysis.           |
| Often includes cleaning and normalization.       | Focuses on vocabulary, token formation, and word normalization. |
| Goal: Prepare text for NLP.                      | Goal: Convert text into meaningful lexical units.               |

## Syntatic Analysis

Syntactic Analysis (also called **Parsing**) is the process of analyzing the **grammatical structure of a sentence** to understand how words are organized and related according to the rules of a language.

Its goal is to answer questions such as:
  - * Which word is the subject?
  - * Which word is the verb?
  - * Which words form a noun phrase?
  - * What is the object of the sentence?

Example Sentence

```text
Microsoft announced Copilot at the Build conference.
```
### 1. Part-of-Speech (POS) Tagging

**Purpose**

Assigns a grammatical category to each word.


| Word       | POS Tag           |
| ---------- | ----------------- |
| Microsoft  | Proper Noun (NNP) |
| announced  | Verb (VBD)        |
| Copilot    | Proper Noun (NNP) |
| at         | Preposition (IN)  |
| the        | Determiner (DT)   |
| Build      | Proper Noun (NNP) |
| conference | Noun (NN)         |

### Output

```text
Microsoft/NNP
announced/VBD
Copilot/NNP
at/IN
the/DT
Build/NNP
conference/NN
```

**Use Cases**

* Grammar checking
* Machine translation
* Text parsing
* Information extraction

### 2. Dependency Parsing

**Purpose**

Determines **how words depend on one another**.

Instead of grouping words into phrases, it builds **head-dependent relationships**.

Sentence

```text
Microsoft announced Copilot at the Build conference.
```

Dependency relationships

```text
announced
├── Subject → Microsoft
├── Object  → Copilot
└── Modifier → conference
                    │
                    ├── at
                    └── Build
```

Or simply

| Head       | Relation | Dependent  |
| ---------- | -------- | ---------- |
| announced  | Subject  | Microsoft  |
| announced  | Object   | Copilot    |
| announced  | Modifier | conference |
| conference | Compound | Build      |

**Use Cases**

* Question Answering
* Information Extraction
* Knowledge Graphs
* Relation Extraction
* Chatbots



### 3. Constituency Parsing

Groups words into **phrases (constituents)** according to grammar.

Instead of focusing on word-to-word dependencies, it identifies phrase structure.

Example

Sentence

```text
Microsoft announced Copilot at the Build conference.
```

Tree

```text
Sentence (S)
├── Noun Phrase (NP)
│     └── Microsoft
└── Verb Phrase (VP)
      ├── announced
      ├── Noun Phrase (NP)
      │      └── Copilot
      └── Prepositional Phrase (PP)
             ├── at
             └── Noun Phrase (NP)
                    ├── Build
                    └── conference
```

**Use Cases**

* Grammar checking
* Machine translation
* Sentence generation
* Educational NLP

### Comparison

| Technique                | What it Analyzes            | Output          | Example                            |
| ------------------------ | --------------------------- | --------------- | ---------------------------------- |
| **POS Tagging**          | Individual words            | Word categories | `announced → Verb`                 |
| **Dependency Parsing**   | Relationships between words | Dependency tree | `Microsoft → subject of announced` |
| **Constituency Parsing** | Groups of words (phrases)   | Phrase tree     | `Microsoft = Noun Phrase`          |

#### How They Work Together

```text
Sentence
    │
    ▼
POS Tagging
    │
    ▼
Dependency Parsing
    │
    ▼
Constituency Parsing
```

Although shown sequentially here for learning, in practice these analyses are often produced independently or by the same parser.

#### Easy Way to Remember

* **POS Tagging** → **What is each word?** *(Noun, Verb, Adjective, etc.)*
* **Dependency Parsing** → **How are the words connected?** *(Subject, Object, Modifier, etc.)*
* **Constituency Parsing** → **Which words belong together as phrases?** *(Noun Phrase, Verb Phrase, Prepositional Phrase, etc.)*

This progression mirrors increasing levels of grammatical understanding, from **word-level categories**, to **word relationships**, to **sentence phrase structure**.
