# Text Preprocessing in Natural Language Processing (NLP)

Text preprocessing is the first stage of every Natural Language Processing (NLP) pipeline. Raw text collected from websites, documents, emails, social media, or user input is often inconsistent, noisy, and unstructured. Preprocessing transforms this raw text into a clean, standardized, and machine-readable format, improving the performance, accuracy, and efficiency of downstream NLP tasks.

Whether building a search engine, chatbot, sentiment analysis model, Retrieval-Augmented Generation (RAG) system, or Large Language Model (LLM) application, effective text preprocessing is essential.



# Objectives

* Convert raw text into a standardized format.
* Remove noise and irrelevant information.
* Reduce vocabulary size.
* Preserve meaningful linguistic information.
* Improve model accuracy and retrieval quality.
* Prepare text for feature extraction and embedding generation.

# 1. Unicode & Encoding Handling

## Description

Text collected from different sources may use different character encodings (UTF-8, ASCII, UTF-16, ISO-8859-1, etc.). Proper encoding ensures all characters are correctly interpreted.

Unicode normalization also standardizes visually identical characters.



### Example

**Input**

```text
Café
```

UTF-8 Representation

```text
Café
```

Unicode Normalized

```text
Cafe
```



## Common Techniques

* UTF-8 conversion
* Unicode normalization (NFC, NFD, NFKC, NFKD)
* Accent removal
* Emoji handling
* Character encoding detection



## Applications

* Multilingual NLP
* OCR
* Web scraping
* Search engines



# 2. Text Cleaning

## Description

Removes unwanted or irrelevant elements from raw text.



### Example

Input

```text
Dr. John Smith's email is John.Smith@Example.com! 😊
```

After Cleaning

```text
Dr John Smith email is John Smith Example com
```



## Common Techniques

* Remove HTML tags
* Remove URLs
* Remove Email Addresses
* Remove Special Characters
* Remove Emojis
* Remove Hashtags
* Remove Mentions (@username)
* Remove Extra Spaces
* Remove Repeated Characters
* Remove Numbers (optional)
* Remove Punctuation (optional)



## Applications

* Social media analytics
* Customer reviews
* Search indexing



# 3. Text Normalization

## Description

Converts text into a consistent format.



### Example

Input

```text
I'm HAPPY!!!
```

Output

```text
i am happy
```



## Common Techniques

* Lowercasing
* Uppercasing (when required)
* Expand contractions
* Standardize abbreviations
* Normalize whitespace
* Standardize spelling
* Convert numbers to words (optional)
* Remove punctuation



## Applications

* Chatbots
* Search
* Text classification



# 4. Sentence Segmentation

## Description

Splits a paragraph into individual sentences.



### Example

Input

```text
John visited New York. He met Microsoft engineers.
```

Output

```text
Sentence 1:
John visited New York.

Sentence 2:
He met Microsoft engineers.
```



## Applications

* Summarization
* Machine Translation
* Question Answering



# 5. Tokenization

## Description

Splits sentences into smaller units called tokens.



### Word Tokenization

Input

```text
Microsoft launched Copilot.
```

Output

```text
["Microsoft",
 "launched",
 "Copilot"]
```



### Subword Tokenization

```text
["Micro",
 "##soft",
 "launch",
 "##ed"]
```

Used by:

* BERT
* GPT
* RoBERTa



### Character Tokenization

```text
["M",
 "i",
 "c",
 "r",
 "o",
 ...]
```



## Tokenization Types

* Word
* Sentence
* Character
* Subword
* Byte Pair Encoding (BPE)
* WordPiece
* SentencePiece



## Applications

Every NLP system.



# 6. Stop Word Removal

## Description

Stop words are common words that often contribute little semantic meaning.

Examples

```text
the
is
a
an
of
to
in
```



### Example

Input

```text
Microsoft launched the Copilot in Seattle.
```

Output

```text
Microsoft launched Copilot Seattle
```



## Advantages

* Smaller vocabulary
* Faster processing
* Better keyword search



## When NOT to Remove

* Large Language Models
* Machine Translation
* Text Generation
* Question Answering

Because stop words often carry grammatical meaning.



# 7. Stemming

## Description

Reduces words by removing prefixes or suffixes using simple rules.

The output may **not** be a valid dictionary word.



### Example

| Original   | Stem    |
| ---------- | ------- |
| Running    | run     |
| Studies    | studi   |
| Connection | connect |
| Flying     | fli     |



## Popular Algorithms

* Porter Stemmer
* Snowball Stemmer
* Lancaster Stemmer



## Applications

* Search Engines
* Information Retrieval
* Keyword Matching



# 8. Lemmatization

## Description

Converts words into their dictionary base form (lemma) using vocabulary and grammar.

Unlike stemming, the output is a valid word.



### Example

| Original | Lemma |
| -------- | ----- |
| Running  | Run   |
| Better   | Good  |
| Children | Child |
| Was      | Be    |



## Advantages

* More accurate
* Preserves meaning
* Better semantic understanding



## Applications

* Semantic Search
* NLP Research
* Chatbots
* Question Answering



# Stemming vs Lemmatization

| Feature             | Stemming       | Lemmatization |
| ------------------- | -------------- | ------------- |
| Uses Grammar        | ❌              | ✅             |
| Produces Valid Word | ❌              | ✅             |
| Speed               | Faster         | Slower        |
| Accuracy            | Lower          | Higher        |
| Best For            | Search Engines | Modern NLP    |

# Key Takeaways

* **Unicode & Encoding Handling** ensures text is interpreted consistently across languages and systems.
* **Text Cleaning** removes noise such as HTML, URLs, emojis, and unwanted symbols.
* **Normalization** standardizes text into a consistent format.
* **Sentence Segmentation** divides paragraphs into meaningful sentence units.
* **Tokenization** converts text into tokens that NLP models can process.
* **Stop Word Removal** reduces vocabulary for traditional NLP, but is generally avoided in LLM pipelines.
* **Stemming** is fast and useful for keyword-based retrieval but may produce non-dictionary words.
* **Lemmatization** preserves grammatical meaning and is preferred for semantic understanding.

Together, these preprocessing techniques form the foundation of robust NLP systems, enabling more accurate information retrieval, text classification, semantic search, question answering, and modern LLM-powered applications.
