# Word2Vec — How It Actually Learns (Intuitive Walkthrough)

## Overview

This README explains Word2Vec not as a formula, but as a **"fill-in-the-neighborhood game."** The model starts knowing nothing about words — it only ever sees raw text. Through millions of small, repeated corrections, it gradually discovers that words appearing in similar contexts should end up with similar numerical representations. Nothing about grammar or meaning is ever explicitly taught — it **emerges** purely from prediction and error correction.


## The Learning Process, Step by Step

### Step 1 — Start with a Text Corpus

The model is given raw sentences with no labels, no definitions, no grammar rules:

```
The king rules the kingdom.
The queen rules the kingdom.
The prince rules the kingdom.
The princess rules the kingdom.
```

At this point, the model has no idea what "king," "queen," or "kingdom" mean — it only sees a stream of tokens.

### Step 2 — Generate Training Examples (Target–Context Pairs)

Using a **window size of 2**, every word in a sentence becomes a target, paired with the words around it:

```
Sentence: "The king rules the kingdom."
```

| Target Word | Context Words |
|---|---|
| king | The, rules |
| rules | king, the |
| kingdom | the, rules |

This process runs automatically across the entire corpus, producing millions (or billions) of such pairs — no human labels required.

### Step 3 — Initialize Random Vectors

Every word starts with a completely random vector — these numbers mean nothing yet:

```
King  → [0.31, -0.44, 0.12]
Queen → [-0.76, 0.11, -0.58]
Apple → [0.93, 0.40, -0.18]
Dog   → [-0.10, 0.71, 0.24]
```

### Step 4 — Make a Prediction

Given the phrase:

```
"The king ___ the kingdom"
```

The model tries to predict the missing word. Since all vectors are still random, early predictions are essentially guesses — it might output "apple," "car," or "dog" instead of the correct word, "rules."

### Step 5 — Measure the Error

The model compares its guess to the actual word that appeared in the corpus:

```
Expected : rules
Predicted: dog
```

This mismatch produces an **error signal** that tells the model exactly how wrong it was.

### Step 6 — Update the Vectors (Backpropagation)

Using gradient descent, the model nudges the vectors for "king," "rules," and "kingdom" slightly closer together, while pushing unrelated words like "apple" further away:

```
Before:  King ------------------------- Apple
After:   King ---- Rules ---- Kingdom
         Apple ----------------------```

Each individual update is tiny — the real learning comes from repetition at scale.

### Step 7 — Repeat, Millions of Times

```
Sentence 1 → Update → Sentence 2 → Update → Sentence 3 → Update → ... → Billions of updates
```

Over enough repetitions, the random noise settles into stable, structured vectors.


## Why Similar Words End Up With Similar Vectors

Consider this small corpus:

```
Cats drink milk.
Dogs drink milk.
Cats chase mice.
Dogs chase balls.
```

Both "cats" and "dogs" repeatedly appear next to the same words — "drink," "milk," "chase." Because the model is optimized to predict those same neighbors correctly for both words, it's mathematically pushed to assign "cats" and "dogs" **similar vectors** — even though nobody ever told it these are both animals.


## Visual Intuition: Before and After Training

**Before training** — vectors are scattered randomly, with no structure:

```

```

Royal titles cluster together, animals cluster together, and vehicles cluster together — purely as a side effect of shared context, not because any category label was ever provided.


## What the Neural Network Is Actually Learning

The network itself is small and shallow:

```
        Target Word
             │
             ▼
      Input Vector
             │
     (Embedding Matrix)
             │
             ▼
      Hidden Layer
             │
             ▼
  Predict Context Words
```

The **weights between the input and hidden layer** are the real prize — after training, that weight matrix *is* the word embedding table:

```
Embedding Matrix
King  → [0.41, -0.32, 0.18, ...]
Queen → [0.43, -0.30, 0.16, ...]
Dog   → [-0.51, 0.74, -0.12, ...]
```

Everything else in the network (the output prediction layer) is discarded once training finishes — only these learned vectors are kept and reused.


## The Governing Principle

> **Words that appear in similar contexts should end up with similar vector representations.**

Word2Vec implements this principle through one of two prediction tasks:

| Architecture | Task | Direction |
|---|---|---|
| **Skip-gram** | Given a target word, predict its surrounding context words | 1 word → many words |
| **CBOW** (Continuous Bag of Words) | Given the surrounding context words, predict the target word | many words → 1 word |

Solving either task correctly, millions of times over, forces the model to discover semantic relationships — without ever being explicitly taught grammar, categories, or meaning.


## Complete Learning Pipeline

```
Large Text Corpus
        │
        ▼
Generate Target–Context Pairs
        │
        ▼
Initialize Random Word Vectors
        │
        ▼
Predict Context (Skip-gram) or Predict Target (CBOW)
        │
        ▼
Calculate Prediction Error
        │
        ▼
Update Vectors via Backpropagation
        │
        ▼
Repeat Millions of Times
        │
        ▼
Learn Meaningful Word Embeddings
```


## Why This Counts as Self-Supervised Learning

Word2Vec never touches a manually labeled dataset. The **text itself provides the supervision**: the words that happen to sit next to each other in real sentences serve as the "correct answers" for the prediction task. This is what makes it **self-supervised** rather than supervised — labels are extracted automatically from the structure of the raw data itself, which is why Word2Vec could scale to enormous, freely available corpora like Wikipedia and news archives without any human annotation effort.

## Key Takeaway

Word2Vec doesn't "know" what a king or a queen is. It never sees a dictionary, an ontology, or a labeled example. All it does is get corrected, over and over, for guessing the wrong neighboring word — and out of that repeated correction, a **geometric map of meaning** emerges, where distance and direction between vectors reflect real semantic relationships.
